- ABSTRACT_FRAME_VARIATION_V1 のあと、人生系 follow-up が R22_ESSENCE_ASK_V1 / R22_COMPARE_ASK_V1 に落ちても centerKey=life を保持する

- 「人生とは？」は ABSTRACT_FRAME_VARIATION_V1 / centerKey=life で返る
- その後の「要するに？」「本質は？」「比較すると？」は
  - R22_ESSENCE_ASK_V1 / centerKey=null
  - R22_COMPARE_ASK_V1 / centerKey=null

  - center_type=concept
  - center_key=''（空）
  - source_route_reason=R22_COMPARE_ASK_V1
  に上書きされている
- つまり abstract define 後に center が保存されていないか、ask系が null で上書きしている

       centerLabel: __abstractPickDefTop.centerLabel,

B. __threadCenterForGeneral の読込を「DB優先 + threadCore fallback」にする
chat.ts の FOLLOWUP_BLOCK 先頭付近:
既存:

これを次へ置換:
if (c && (c.center_key || (__threadCore.centerKey && c.center_type === "concept"))) {
  __threadCenterForGeneral = {
    center_type: c.center_type,
    center_key: String(c.center_key || __threadCore.centerKey || "").trim(),
    source_route_reason: c.source_route_reason ?? undefined,
  };

    center_type: "concept",

  };
}
