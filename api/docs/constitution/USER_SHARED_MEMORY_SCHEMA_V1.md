# USER_SHARED_MEMORY_SCHEMA_V1

**MODE:** `DOCS_ONLY`（runtime 改修なし）  
**カード:** `USER_SHARED_MEMORY_SCHEMA_V1`  
**上位文書:** `SELF_BUILD_GOVERNOR_V1.md`  
**no-touch:** `api/src/db/kokuzo_schema.sql`（本スキーマの **物理保存先・DDL は別カード**で定義。本書は論理契約のみ）  
**関連:** デバイスローカル側の論理スキーマは **`DEVICE_LOCAL_CACHE_SCHEMA_V1`**（境界の相手方）

---

## 1. 目的

同一 **ログイン・ユーザー** が複数デバイスから接続する際、**個人の記憶・人格・知識設定**を共有するための **サーバ側（または user スコープの正本）** の論理スキーマを固定する。  
**他ユーザーのデータと混ざらない**ことを最優先の設計制約とする。

---

## 2. 識別子と分離原則

| 原則 | 内容 |
|------|------|
| **user 単位分離** | すべての共有レコードは **`user_id`**（またはそれと 1:1 の安定 id）に **必ず紐づく**。クエリ・同期・export は **常に user スコープで開始**する。 |
| **device は user 配下** | `device_id` は **user 内**でのみ一意。`(user_id, device_id)` の組で端末を指す。共有スキーマの「正本」は **device に依存しない** フィールドが主。 |
| **混線禁止** | レスポンス合成・同期・バッチは **user_id をキーにしたパス**のみを通す。他 user の id をクロス参照しない。 |

---

## 3. 論理スキーマ（最低項目）

実装時の JSON 名・DB 列名は任意だが、**意味は次のブロックを欠かない**。

### 3.1 `user_id`

- ログイン主体の安定識別子（内部 UUID 等）。**すべての共有レコードの第一キー。**

### 3.2 `naming_profile`

- AI 呼称・ユーザー呼称・二人称・敬体/常体など **命名・呼び方** の設定。
- 他ユーザーとの共有は行わない。

### 3.3 `persona_profile`

- 人格核・口調・禁止トーン・役割（会話上の「枠」）**の user スコープ設定**。
- 正典・ルール文書との衝突は別カード（Governor）で裁定するが、**保存値は user 単位**。

### 3.4 `response_style_profile`

- 短め／長め／厳密／柔らかい等の **返答スタイル** 設定。
- 端末ごとの一時的な上書きは **DEVICE_LOCAL_CACHE** 側に寄せる（`DEVICE_LOCAL_CACHE_SCHEMA_V1`）。

### 3.5 `memory_inheritance_profile`

- 旧カスタム GPT 等から継承した **長文プロンプトの分解結果**（または正規化済みフィールド）を保持するスロット。
- 詳細フィールドは **`MEMORY_INHERITANCE_PROFILE_SCHEMA_V1`** で拡張する。

### 3.6 `shared_memory_facts`

- 複数デバイスで共有したい **事実・好み・前提** のリスト（またはバージョン付き集合）。
- 重い推論・学習の中間表現は **ここに持たない**（`local_only` へ）。  
- 同期・マージ規則は **`USER_DEVICE_MEMORY_SYNC_POLICY_V1`** / **`USER_DEVICE_MEMORY_SYNC_ENGINE_V1`**。

### 3.7 `source_connector_settings`

- Notion / NotebookLM / 外部 URL 等の **コネクタ設定**（トークン・参照 id は **user に紐づく vault** に保存し、平文ログに出さない）。
- コネクタの **user binding** は **`USER_MEMORY_ISOLATION_GUARD_V1`** で固定。

### 3.8 `growth_log_pointer`

- 成長・学習・自己構築ログの **参照先**（例: `artifact_manifest`、ledger パス、外部オブジェクトキー）。**実体はポインタのみ**とし、VPS 負荷を抑える。
- 重い学習処理は端末ローカルに残す方針と整合させる（実体は local または別バケット）。

---

## 4. device local cache との役割分離

| 層 | 保持するもの | 本スキーマとの関係 |
|----|----------------|-------------------|
| **共有（本書）** | 上記 3.1〜3.8 の **user 正本** | 同一 user の全デバイスが **読み取り・更新の対象**（同期ポリシーに従う） |
| **ローカル（別紙）** | 埋め込み・大きな一時要約・未同期ドラフト・キュー | **`DEVICE_LOCAL_CACHE_SCHEMA_V1`**。サーバへ上げるのは **ポリシーで許可された差分のみ** |

**ルール:** 共有スキーマに **生の embedding ベクトル**や **巨大な未確定ドラフト** を詰め込まない（負荷・漏洩・混線リスク）。

---

## 5. 他ユーザーとの混線防止（方針）

- すべての API・同期ジョブは **認可済み `user_id` からのみ** データを読み書きする。
- export / import は **別カード**で **明示承認・監査ログ** を必須とする（本書では要件のみ）。
- クロスユーザー import は **禁止**（`USER_MEMORY_ISOLATION_GUARD_V1`）。
